# Research: Integridade do Campo texto_bruto

**Date**: 2026-05-07

## 1. Executive Summary
A corrupção do campo `texto_bruto` no dashboard (exibindo usernames em vez de comentários) foi rastreada até o seletor DOM vago utilizado no `core/instagram_headless.py`. O sistema captura indevidamente o nome do autor devido a mudanças na estrutura do Instagram.

## 2. Technical Context
- **Arquivo Afetado**: `core/instagram_headless.py:169`
- **Implementação Atual**: 
  ```python
  comments = await self.page.evaluate("() => Array.from(document.querySelectorAll('div[role=\"button\"] span')).map(s => s.innerText).filter(t => t.length > 5)")
  ```
- **Comportamento Atual**: O seletor `div[role="button"] span` captura todos os spans dentro de botões/links, o que no novo layout do Instagram inclui o nome de exibição do autor do comentário. O filtro `length > 5` não impede que nomes de usuários sejam capturados.

## 3. Findings & Analysis
- **Seletor Alternativo Identificado**: No arquivo `core/stealth_playwright.py:75`, utiliza-se:
  ```python
  elements = await page.query_selector_all('div.x9f619 span[dir="auto"]')
  ```
- **Análise do DOM**: O Instagram utiliza a classe `x9f619` para containers de comentários e `span[dir="auto"]` para o conteúdo textual. O seletor em `instagram_headless.py` é uma simplificação perigosa que faliu após atualizações da plataforma.
- **Frontend Fallback**: O arquivo `src/core/ui.js:272` já possui uma lógica de fallback, mas como o `texto_bruto` está sendo preenchido com o username (que não é vazio), o fallback nunca é acionado.

## 4. Technical Constraints
- O scraper `instagram_headless.py` roda em modo headless e não utiliza perfis persistentes por padrão, dependendo de seletores altamente estáveis.
- O `TextProcessor` (`processing/text_processor.py`) está praticamente vazio, o que impede qualquer limpeza ou validação pós-coleta antes da inserção no banco.

## 5. Architecture Documentation
- O fluxo de dados segue o padrão: `HeadlessScraper` -> `Supabase (comentarios)` -> `API` -> `UI`.
- A falta de uma camada de validação no `TextProcessor` é um ponto cego arquitetural.
