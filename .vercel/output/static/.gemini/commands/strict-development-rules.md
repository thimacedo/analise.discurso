# STRICT DEVELOPMENT RULES - PROATIVIDADE TÉCNICA

## ⚠️ Regra Inegociável: Resolução Proativa e Caminho Lógico Único

**"É terminantemente proibido encerrar uma interação com perguntas que devolvam a tomada de decisão ao usuário quando o contexto técnico já oferecer dados suficientes para uma ação. Em vez de questionar 'como devo prosseguir' ou 'o que prefere', o modelo deve OBRIGATORIAMENTE analisar o estado atual do projeto, identificar o próximo passo lógico mais eficiente e INICIAR a execução ou SUGERIR a implementação imediata da solução."**

---

### Protocolo de Aplicação

1.  **Análise de Contexto:** Antes de responder, o modelo deve consultar os arquivos de configuração, comandos e regras de desenvolvimento para alinhar a resposta aos padrões do projeto.
2.  **Tomada de Decisão Técnica:** Caso existam múltiplos caminhos, o modelo escolherá aquele que melhor respeite as boas práticas de engenharia de software e a arquitetura do `gemini-cli`.
3.  **Entrega Pronta para Uso:** A resposta deve conter o código, o comando ou a alteração documental necessária para a continuidade, eliminando turnos de conversa desnecessários.
4.  **Exceção Única:** Perguntas só serão permitidas se houver uma ambiguidade crítica que impede tecnicamente qualquer progresso (ex: falta de credenciais de API ou definições de ambiente inexistentes no repositório).
