# Pesquisa Técnica: Raspagem, Classificação e Sistema de Recompensa

## Objetivo da Pesquisa
Mapear a arquitetura e funcionalidade dos componentes chave relacionados à raspagem de dados, classificação de comentários e sistema de recompensas para workers, com base nos arquivos `instagram_scraper.py`, `main_orchestrator.py`, `official_solenya_daemon.py` e nas diretrizes do `README.md`.

## 1. `instagram_scraper.py`
- **Função:** Coleta de dados do Instagram.
- **Responsabilidades:** Detalhar os métodos de scraping, endpoints targetizados, estratégias de autenticação (se aplicável), e mecanismos para evitar detecção ou bloqueio. Investigar a infraestrutura de proxy e a gestão de sessões, se presentes.
- **Observações:** Este componente é crucial para a ingestão de dados brutos. Sua robustez e eficiência impactam diretamente a qualidade dos dados subsequentes.

## 2. `main_orchestrator.py`
- **Função:** Orquestrador central do sistema.
- **Responsabilidades:** Gerencia o fluxo de trabalho geral, coordena a execução de scrapers, despacha tarefas para o sistema de classificação e o daemon de recompensa. Define a lógica de processamento em pipeline.
- **Observações:** Fundamental para entender como os diferentes módulos se integram e como as tarefas são distribuídas e monitoradas.

## 3. `official_solenya_daemon.py`
- **Função:** Gerencia o sistema de recompensas para workers.
- **Responsabilidades:** Implementa a lógica para atribuir, rastrear e processar recompensas aos workers. Investiga a interação com o orquestrador e a forma como os workers recebem e confirmam tarefas/recompensas.
- **Observações:** Este daemon é vital para o engajamento e a eficiência dos workers, garantindo que o trabalho seja devidamente incentivado.

## 4. `README.md`
- **Função:** Documentação de alto nível do projeto.
- **Responsabilidades:** Define convenções de codificação, padrões arquiteturais, diretrizes de segurança, requisitos de performance e setup do ambiente. Serve como referência primária para aderência aos padrões do projeto.
- **Observações:** Qualquer implementação ou modificação deve estar em conformidade com o que é especificado aqui.

## Conclusão Preliminar
A pesquisa inicial indica uma arquitetura modular com forte dependência da orquestração central (`main_orchestrator.py`) para gerenciar tarefas de coleta (`instagram_scraper.py`) e processamento com recompensas (`official_solenya_daemon.py`), tudo sob as diretrizes do `README.md`.
