# Implementation Plan: Correção de Integridade de Comentários (tkt_fix_text_1)

## Overview
Corrigir a falha de captura de dados onde o nome do autor é salvo no campo `texto_bruto` em vez do comentário real.

## Scope Definition (CRITICAL)
### In Scope
- Correção do seletor DOM em `core/instagram_headless.py`.
- Implementação de lógica de validação em `processing/text_processor.py`.
- Adição de fallback no frontend em `src/core/ui.js`.
### Out of Scope (DO NOT TOUCH)
- Geração do mapa de arquitetura (Ticket #2).
- Mudança de provedor de IA ou scrapers alternativos.

## Current State Analysis
- **Bug**: `core/instagram_headless.py:169` usa seletor vago `div[role="button"] span`.
- **Slop**: `processing/text_processor.py` está vazio.
- **Frontend**: `src/core/ui.js` exibe o que estiver no `texto_bruto` se não for nulo.

## Implementation Phases

### Phase 1: Scraper Fix (core/instagram_headless.py)
- **Goal**: Garantir a extração do texto real.
- **Steps**:
  1. [ ] Atualizar o `page.evaluate` para usar seletores mais resilientes: `span._ap30` ou `div.x9f619 span[dir="auto"]`.
  2. [ ] Adicionar um `console.log` interno no evaluate para depuração se necessário.
- **Verification**: Rodar o scraper localmente e inspecionar o objeto `data` antes da inserção.

### Phase 2: Data Guardrail (processing/text_processor.py)
- **Goal**: Impedir que lixo (usernames) chegue ao banco.
- **Steps**:
  1. [ ] Implementar a função `clean_comment(text, author_username)`.
  2. [ ] Se `text.strip() == author_username.strip()`, retornar `None`.
  3. [ ] Integrar esta chamada no `core/instagram_headless.py` antes do `supabase.insert`.
- **Verification**: Teste unitário manual com strings idênticas.

### Phase 3: UI Resilience (src/core/ui.js)
- **Goal**: Fallback para dados históricos corrompidos.
- **Steps**:
  1. [ ] Modificar a linha 272 de `src/core/ui.js`.
  2. [ ] Adicionar condição: `(alerta.texto_bruto !== alerta.autor_username)` antes de exibir o texto bruto.
- **Verification**: Recarregar o dashboard e verificar se o fallback "Conteúdo indisponível..." aparece para os registros afetados.
