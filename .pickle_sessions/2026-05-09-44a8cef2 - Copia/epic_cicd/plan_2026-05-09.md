# CI/CD Quality Gates Implementation Plan

## Overview
Implementar um pipeline robusto de CI/CD no GitHub Actions focado em garantir a qualidade e estabilidade da branch `main`. Este pipeline rodará exclusivamente em Pull Requests destinados à `main` e validará o código fonte através de linters estritos, criados do zero para barrar más práticas e garantir o God Mode padrão.

## Scope Definition (CRITICAL)
### In Scope
- Criação de workflow do GitHub Actions (`.github/workflows/ci.yml`).
- Configuração estrita do Ruff e Black (para backend Python).
- Configuração estrita do ESLint (para frontend JavaScript).
- O pipeline bloqueia ativamente o PR se o código não for compatível com as regras.
### Out of Scope (DO NOT TOUCH)
- Implementar testes automatizados (isso não faz parte da task de linters, embora o CI futuramente vá chamá-los).
- Alterar as lógicas dos scrapers existentes ou o banco de dados.
- Alterar o `vite` ou build assets do frontend (isso pertence à tarefa de Bundler).

## Current State Analysis
Atualmente, não há um pipeline unificado definido em `.github/workflows/ci.yml` para bloqueios e o código pode ser mergeado sem verificações estáticas, permitindo "AI slop" e quebra de regras.

## Implementation Phases
### Phase 1: Setup das Configurações Iniciais (Linters)
- **Goal**: Definir as regras estritas e inflexíveis.
- **Steps**:
  1. [x] Criar `pyproject.toml` (ou atualizar se já existir) com regras agressivas do Ruff (ignorar E501 porque Black cuida disso, focar em C, E, F, W, B).
  2. [x] Criar `.eslintrc.json` configurando regras de null-safety obrigatórias, zero `any` ou var (focar em let/const e pureza de ESModules).
- **Verification**: Rodar `npx eslint .` e `ruff check .` localmente e garantir que consigam ler as configs geradas.

### Phase 2: Construção do Workflow CI/CD
- **Goal**: O GitHub deve orquestrar as verificações a cada Pull Request.
- **Steps**:
  1. [x] Criar arquivo `.github/workflows/ci.yml`.
  2. [x] Configurar gatilho: `on: pull_request: branches: [main]`.
  3. [x] Criar Job de Backend: Instalar Python, instalar dependências `requirements.txt`, rodar `black --check .` e `ruff check .`.
  4. [x] Criar Job de Frontend: Instalar Node.js, `npm ci`, rodar `npx eslint .`.
- **Verification**: Simular (se possível) a execução ou apenas commit e visualização se os steps do github validaram a sintaxe do Yaml (`gh workflow view` ou checagem manual criteriosa).
