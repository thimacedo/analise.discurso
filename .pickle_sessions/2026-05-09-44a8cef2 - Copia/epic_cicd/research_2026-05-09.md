# Research: CI/CD Quality Gates

**Date**: 2026-05-09

## 1. Executive Summary
The codebase currently lacks strict CI/CD quality gates. There is a `package.json` present, but no robust `.github/workflows/ci.yml`, `pyproject.toml`, or `.eslintrc.json`. Code can be merged without static analysis, allowing "AI slop" to enter the `main` branch.

## 2. Technical Context
- `package.json` exists (verified via glob).
- Python backend has no explicitly found `pyproject.toml` in the recent glob, implying we need to create or heavily modify it.
- No `ci.yml` exists in `.github/workflows/`.

## 3. Findings & Analysis
The plan (`plan_2026-05-09.md`) outlines two clear phases:
1. **Setup Linters**: Create `pyproject.toml` for Ruff/Black and `.eslintrc.json` for ESLint. Focus on null-safety and no `any`.
2. **CI/CD Pipeline**: Create `.github/workflows/ci.yml` that runs on PRs to `main`. It will trigger Ruff, Black, and ESLint checks.
The environment uses Python and Node.js.

## 4. Technical Constraints
- The pipeline MUST block PRs that fail linting.
- No automated tests are in scope for this ticket.
- Do not modify existing logic or Vite build configurations.

## 5. Architecture Documentation
Current setup: Loose, no strict enforcement.
Target setup: Rigid GitHub Actions pipeline enforcing Ruff, Black, and ESLint.
